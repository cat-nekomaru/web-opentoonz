# .TLW2 - Sequential Image File Format

**OpenToonz-Web** に最適化した独自のセル画フォーマット（type-2）。

`.tlv`（OpenToonz標準）の設計が複雑かつ古いため、独自路線で設計することにした。  
`.tlw`（ZIPベースの本命フォーマット）を作る前に、**高速検証・テスト用**として先に`.tlw2`の仕様を固めることになった。

## 基本コンセプト

- **固定長ヘッダー（1024B）** + **サムネイル先行（512Bアライメント）** + **可変長Index Table** + **本番PNG群** + **任意フッター（アライメント対象外）**
- **高速表示と障害耐性**：冒頭の数KBをシークするだけでループ処理による超高速プレビューが可能。万が一のデータ破損時もサムネイルを確実に救出できる設計。
- 連番PNG-8（線画・塗りを別管理可）をバイナリ埋め込み
- 主な用途：**テストコード・高速プレビュー・ファイルI/O検証**

## 特徴・トレードオフ

**Good**
- 専用ツール上での高速読み書き
- Web版からのエクスポート／インポートが比較的容易
- ヘッダーにコメントを残せる（デバッグ・設計メモに便利）
- ファイルI/Oの練習・検証に最適

**Not Good**
- OS標準のファイルビューアで中身が見えない
- 大量のセル（数百以上）には不向き
- 将来的には廃止 or 補助フォーマットになる可能性が高い

## ファイル構造

※[Header]から[PNG Data Blocks]までは、各セクションおよび各画像データの開始位置がすべて512バイト境界（アライメント）に配置されます。

```text
[Header]          1024バイト固定（ASCII/Unicodeテキスト）
[Preview Blocks]  512バイトアライメント（可変長, バイナリ）
[Index Table]     512バイトアライメント（可変長, ASCIIテキスト）
[PNG Data Blocks] 512バイトアライメント（可変長）
[Footer]          任意（アライメント対象外 / 使用例：シーン.xml, パレット.xmlなど）
```
